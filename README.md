# MANTIS GNN Stocks

MANTIS GNN Stocks builds a temporal financial ecosystem graph and exports a MANTIS-ready CSV for representation learning and manifold/cartographic analysis. Instead of treating each stock as an isolated time series, the project models markets as a connected system of equities, sectors, ETFs, countries, commodities, currencies, macro pressure, geopolitical events, regulation, and news narratives.

The goal is not stock picking or trading advice. The goal is to create a rich `(entity, date)` dataset where every row combines interpretable descriptors with learned graph embeddings.

## What This Project Produces

The pipeline writes a CSV like:

```text
entity,date,<financial features>,<world-event features>,<graph features>,gnn_1,...,gnn_N
```

The default sample run writes:

```text
outputs/mantis_financial_ecosystem_dataset.csv
data/sample/sample_output.csv
```

Larger real-feed configs write additional files in `outputs/`, including compressed and under-50MB variants.

Each row contains:

- Financial descriptors: returns, log returns, volatility, RSI, MACD, Bollinger z-score, ATR, beta, Sharpe-style ratios, momentum, drawdown, autocorrelation, entropy, and volatility clustering.
- World-event and macro descriptors: inflation pressure, rate sensitivity, tariff exposure, commodity dependence, geopolitical instability, conflict intensity, regulatory uncertainty, macro surprise, and narrative momentum.
- News/NLP-style descriptors: sentiment, article volume, sentiment variance, narrative novelty, narrative persistence, embedding centroid shift, topic cluster, and geopolitical intensity.
- Graph descriptors: centrality, PageRank, clustering, graph density, neighborhood similarity, ETF exposure, contagion, and local market influence.
- Learned embeddings: `gnn_1` through `gnn_N`, generated by a graph encoder when PyTorch Geometric is installed or by a deterministic SVD fallback otherwise.

See [docs/final_csv_schema.md](docs/final_csv_schema.md) for the column-level schema.

## How It Works From Start To Finish

1. A YAML config chooses the data mode, universe, date range, feature settings, model settings, and export paths.
2. The data loader returns normalized tables for prices, events, news, entity metadata, and ETF holdings.
3. Financial feature code computes rolling market descriptors from OHLCV data.
4. World-event feature code converts events, news, macro stress, and market behavior into interpretable ecosystem signals.
5. The graph builder creates one heterogeneous graph snapshot per date.
6. Graph statistics are computed for every node in every snapshot.
7. The training module learns node embeddings from graph structure and node features.
8. The exporter joins financial features, world-event features, graph statistics, and embeddings into one CSV.

The resulting file is designed for MANTIS or any downstream analysis tool that benefits from both explainable columns and latent graph-representation coordinates.

## Installation

Use Python 3.10 or newer.

```bash
python -m venv .venv
.venv/bin/pip install -r requirements.txt
```

If `torch-geometric` is difficult to install on your machine, the pipeline can still run. It will automatically fall back to deterministic graph-feature SVD embeddings. Install PyTorch Geometric when you want the full GraphSAGE/GAT training path.

## Quickstart: Synthetic Sample

The synthetic mode is the easiest way to verify the full pipeline without API keys or network access.

```bash
PYTHONPATH=src .venv/bin/python scripts/run_sample_pipeline.py --config configs/sample_config.yaml
```

Expected result:

```text
Exported <row count> rows and <column count> columns.
```

The sample config uses a small universe:

```text
NVDA, AMD, MSFT, XOM, JPM, TSLA
```

It creates synthetic prices, events, news, sectors, ETF relationships, countries, commodities, and currencies, then runs the same feature, graph, embedding, and export code used by the real-data modes.

## Real Market OHLCV Mode

Use the yfinance config to pull real historical OHLCV data from Yahoo Finance through `yfinance`:

```bash
PYTHONPATH=src .venv/bin/python scripts/run_sample_pipeline.py --config configs/yfinance_config.yaml
```

This mode uses real market prices. The news, geopolitical, and macro columns are currently proxy descriptors derived from market stress and volume unless you connect dedicated event/news/macro feeds.

## Real Multi-Feed Mode

Use the broad real-feed config for a large equity universe:

```bash
PYTHONPATH=src .venv/bin/python scripts/run_sample_pipeline.py --config configs/real_feeds_950_config.yaml
```

This mode combines:

- Yahoo Finance through `yfinance` for equity, commodity, and currency OHLCV.
- Wikipedia S&P 500 constituents and Nasdaq Trader listed symbols for universe construction.
- FRED CSV endpoints for macro series.
- GDELT 2.1 Doc API for news, geopolitical, and topic signals.

For large universes, correlation graph construction can become expensive. The config supports `features.max_corr_edges_per_node` so each node keeps only its strongest weighted correlation edges.

## Configuration Guide

Pipeline behavior is controlled by YAML files in `configs/`.

Important sections:

- `project`: project name, random seed, and output directory.
- `data`: data mode, dates, tickers, sector mapping, ETFs, countries, commodities, and currencies.
- `features`: rolling windows, graph window, correlation threshold, and embedding dimension.
- `training`: model type, epochs, learning rate, hidden dimension, dropout, negative sampling, temporal consistency weight, and checkpoint path.
- `export`: final CSV path and sample CSV path.

Supported `data.mode` values:

```text
synthetic
yfinance
real_feeds
```

Supported `training.model` values are implemented in `src/mantis_financial_graph/models/gnn.py`.

## The Graph Model

The graph is temporal: each snapshot represents the ecosystem on one date. Nodes can include:

- Stocks
- Sectors
- ETFs
- Countries
- Commodities
- Currencies
- Central banks
- Macro reports
- Geopolitical events
- Regulatory events
- Technology narratives

Edges can represent:

- Rolling stock-to-stock correlation
- Stock-to-sector membership
- ETF ownership exposure
- Technology narrative exposure
- Geopolitical supply-chain exposure
- Tariff exposure
- Commodity dependence
- Geopolitical influence
- Macroeconomic pressure
- Interest-rate sensitivity
- Regulatory impact

Edges carry weights and timestamps. The implementation stores snapshots as NetworkX graphs and converts them to PyTorch Geometric tensors for neural training.

## Embedding Training

When PyTorch Geometric is available, the project trains an unsupervised graph autoencoder-style model:

- Positive examples are observed graph edges.
- Negative examples are sampled non-observed edges.
- A temporal consistency term regularizes embeddings across rolling snapshots.
- The encoder can be GraphSAGE or GAT, depending on the config.

The trained checkpoint is written to the configured `training.checkpoint_path`.

When PyTorch Geometric is not available, the project uses a deterministic fallback:

- Concatenate numeric node features with the graph adjacency matrix.
- Standardize the combined matrix.
- Project it into the configured embedding dimension with truncated SVD.

This fallback keeps the full CSV-generation workflow runnable on lightweight machines.

## Data Contracts

All loaders should normalize their data to the same internal tables:

```text
prices:          entity,date,open,high,low,close,volume,sector
events:          entity,date,event_type,<world-event columns>
news:            entity,date,text,source,sentiment,article_volume,geopolitical_intensity
entity_metadata: entity,node_type,sector,country
etf_holdings:    etf,stock,weight
```

That contract lets synthetic, yfinance, and real-feed modes share the same downstream feature, graph, model, and export code.

## Extending The Project

Add data sources in:

```text
src/mantis_financial_graph/data/
```

Add financial, event, or graph descriptors in:

```text
src/mantis_financial_graph/features/financial.py
src/mantis_financial_graph/features/world_events.py
src/mantis_financial_graph/features/graph_stats.py
```

Add node or edge types in:

```text
src/mantis_financial_graph/graph/builder.py
```

Add or modify graph encoders in:

```text
src/mantis_financial_graph/models/gnn.py
src/mantis_financial_graph/training/train.py
```

## Notes And Limitations

- This is a research dataset pipeline, not a trading system.
- Real-data modes depend on public internet sources and may fail if a provider changes its API, rate limits requests, or returns incomplete data.
- Public news and macro proxies are useful for experimentation, but production-grade market research should use licensed or carefully validated feeds.
- Generated datasets in `outputs/` can be large. Keep the files you want to publish and ignore local scratch outputs.
